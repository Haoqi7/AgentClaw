# SOUL.md 补丁说明 — 产出物目录指引

本文件说明如何为各部门 Agent 的 `SOUL.md` 文件追加「产出物管理」章节，
使 Agent 在执行任务时知道将工作产出保存到统一的目录结构中。

---

## 目录结构约定

所有任务产出物存放在 `data/outputs/{任务ID}/` 下，各部门在该任务目录下
拥有以本部门名称命名的子目录，互不干扰。

示例（任务 ID 为 `JJC-20260223-012`）：

```
data/outputs/JJC-20260223-012/
├── 工部/
├── 兵部/
├── 户部/
├── 礼部/
├── 刑部/
├── 吏部/
├── 中书省/
├── 尚书省/
└── 门下省/
```

## 应用补丁的方法

对于每个目标 `SOUL.md` 文件：

1. 打开文件，定位 `## 看板操作` 这一行。
2. 在该标题下方（下一个空行或正文之前）插入上述「插入内容」中的完整文本。
3. 确保新增的 `## 产出物管理` 章节与前后章节之间保留一个空行。

### 自动化脚本参考（sed）

以工部为例，在 `agents/gongbu/SOUL.md` 中查找 `## 看板操作` 行并在其后插入内容：

```bash
#!/bin/bash
# 用法: ./apply_soul_patch.sh <部门名> <子目录显示名>

DEPT_DIR="$1"
SUBDIR_NAME="$2"
TARGET="agents/${DEPT_DIR}/SOUL.md"

if [ ! -f "$TARGET" ]; then
  echo "错误: $TARGET 不存在"
  exit 1
fi

# 检查是否已经打过补丁
if grep -q '## 产出物管理' "$TARGET"; then
  echo "跳过 $TARGET — 已包含 产出物管理 章节"
  exit 0
fi

# 在 "## 看板操作" 行之后插入
sed -i "/^## 看板操作$/a\\
\\
## 产出物管理\\
\\
任务产出物统一存放于 \\\`data/outputs/{任务ID}/\\\` 目录下。\\
你在执行任务时产生的所有文件（代码、文档、报告、数据等），请保存到该任务目录下以你的部门名称命名的子目录中。\\
\\
例如任务 ID 为 JJC-20260223-012：\\
\\\`\\\`\\\`\\
data/outputs/JJC-20260223-012/${SUBDIR_NAME}/\\
\\\`\\\`\\\`\\
\\
所有部门共享同一个任务目录，各部在各自子目录中工作，互不干扰。\\
" "$TARGET"

echo "已为 $TARGET 打补丁"
```

批量执行：

```bash
./apply_soul_patch.sh gongbu    工部
./apply_soul_patch.sh bingbu    兵部
./apply_soul_patch.sh hubu      户部
./apply_soul_patch.sh libu      礼部
./apply_soul_patch.sh xingbu    刑部
./apply_soul_patch.sh libu_hr   吏部
./apply_soul_patch.sh zhongshu  中书省
./apply_soul_patch.sh shangshu  尚书省
# menxia 需要手动应用（文案不同），或单独编写脚本
```
