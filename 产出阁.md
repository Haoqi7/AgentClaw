# AgentClaw 产出阁 补丁包

## 📦 功能说明

在看板「流程监察」Tab 后新增「产出阁」板块，实现：
- 每个任务的所有产出物集中存放在 `data/outputs/{任务ID}/` 目录下
- 各部门按部门名分子目录存放产出（如 `工部/`、`礼部/`）
- 支持拖拽上传、下载、删除、预览（文本文件）
- 按部门分组展示，部门颜色标签
- 文件类型图标自动识别

## 🔧 安装步骤

### 前端（3 个文件）

1. **复制新组件**：
   ```bash
   cp components/TaskOutputPanel.tsx 项目根目录/edict/frontend/src/components/
   ```

2. **替换 App.tsx**：
   ```bash
   cp App.tsx 项目根目录/edict/frontend/src/App.tsx
   ```
   改动点：仅新增 2 行（import + render），用 `[TaskOutput]` 标注

3. **替换 store.ts**：
   ```bash
   cp store.ts 项目根目录/edict/frontend/src/store.ts
   ```
   改动点：仅新增 1 行 TabKey 和 1 行 TAB_DEFS

4. **替换 api.ts**：
   ```bash
   cp api.ts 项目根目录/edict/frontend/src/api.ts
   ```
   改动点：api 对象新增 3 个方法 + 新增 3 个 Type

5. **重新构建前端**：
   ```bash
   cd edict/frontend && npm run build
   ```

### 后端（2 处修改）

6. **追加 API 到 server.py**：
   - 打开 `dashboard/server.py`
   - 将 `task_output_api.py` 中的函数定义（从 `_OUTPUTS_DIR` 到 `handle_output_delete`）
     追加到 `class Handler` 定义之前
   - 按 `task_output_api.py` 末尾的 `_INSTALL_NOTES` 说明添加路由

7. **修改 kanban_update.py**：
   - 按 `kanban_update.patch.md` 中的说明修改
   - 创建任务时自动初始化产出目录

### Agent 配置（可选）

8. **修改各部门 SOUL.md**：
   - 按 `agents/SOUL_PATCH_INSTRUCTIONS.md` 中的说明修改
   - 让 Agent 知道产出物应该存放在任务目录下

## 🗑️ 回滚方式

所有新增代码都用 `#[TaskOutput]` 或 `// [TaskOutput]` 标注。
回滚只需搜索这些标记并删除对应代码块即可完全移除产出阁功能。
